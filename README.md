Design thinking 
I didn’t set out to make “another drawing web page.” I started by asking: in visual communication, when we study composition rules, modular grids, and axis-based layouts, the hardest part is usually not understanding the rule—it’s turning it into something you can compare visually, over and over. So I framed the product around a short decision chain: import an image → pick a compositional hypothesis → see the guides → tune typographic hierarchy and spacing on the same canvas → export.

At the level of information architecture, I split the interface into three concerns—structure (layers), the artefact (canvas), and decisions (presets and properties). Layers manage order; presets and parameters live on the side; the centre stays for the picture itself, so learning the tool doesn’t steal attention from the visual relationships you’re judging. For guides, I used a soft background grid plus stronger alignment lines on top so the structure stays usable without lines overpowering image and type. Putting rulers, alignment guides, palette, undo, and export in the top bar draws a clear line between global display state and object-level properties, which cuts down on accidental tweaks and mental clutter.

In short: choices like the dark studio look, selectable guide colours, and property panels that respond to selection are all in service of “comparing layouts for a long time”—so people are willing to try more schemes instead of giving up because the tool fights them.
